## Git and Version Control Fundamentals (6 Hours)
Before a student starts building software independently, they must learn how to track, save, and back up their code using industry-standard version control.
* **What to study:** Git installation, basic terminal commands (`git init`, `commit`, `push`, `pull`, `branching`), and managing repositories on GitHub.
* **Free Resource:** GitHub Skills tutorials or the "Introduction to Version Control" modules on Coursera/Kodeco.
* **Why it matters for credit:** It teaches the student how to publish their work online. Their GitHub profile becomes the immutable "lab notebook" that proves to a parent evaluator or college admissions office that they actually wrote the code.

### 2. Google's Intermediate Android Pathways (8 Hours)

Google provides short, highly targeted intermediate pathways that pick up right where *Android Basics with Compose* leaves off.

* **Advanced Architecture & Testing (4 Hours):** Deepens understanding of the separation between the UI layer and data layer using advanced StateFlow and production-level clean code practices.
* **Accessibility & Adaptive Layouts (4 Hours):** Teaches how to optimize apps for diverse form factors (tablets, foldables) and ensure apps are usable for individuals with visual or motor impairments.
* **Where to find it:** Accessible directly via the advanced tabs on the official **[Android Developers Training Portal](https://developer.android.com/courses)**.

### 3. Guided Capstone Portfolio Project (40 Hours)

The ultimate test of programming mastery is transitioning away from video guides to solve abstract problems independently. This 40-hour block should be treated like a high school final exam or science fair project. The student must conceptualize and execute a completely custom Android application from scratch.

To ensure the student stays on track and puts in the required time, structure the 40 hours into clear, accountable phases:

| Phase | Estimated Hours | Student Deliverables |
| --- | --- | --- |
| **Project Scoping & UI Wireframing** | 5 Hours | A written feature list and hand-drawn layout mockups of every app screen. |
| **Core UI Development** | 15 Hours | Building out the structural layout, typography, and theme using Jetpack Compose. |
| **State & Data Architecture** | 10 Hours | Hooking up data models, navigation logic, and integrating local storage (like a Room SQLite database). |
| **Testing & Bug Fixing** | 5 Hours | Writing functional unit tests and resolving logic loops or interface layout bugs. |
| **Documentation & Launch** | 5 Hours | Uploading the final code to GitHub and writing a comprehensive `README.md` explaining how the app works, the technical stack used, and screenshots of the finished product. |

**Excellent Capstone Ideas for High Schoolers:**

* **Personal Habit / Chore Tracker:** An app where users can create daily recurring tasks, check them off, and persist their daily completion "streaks" using local database storage.
* **Flashcard Study Companion:** A customizable quiz app where the student can input custom study questions, categorize them by school subject, and track their test scores over time.
* **Local Wildlife Guide:** A multi-screen app with scrollable lists featuring local flora/fauna, detailed descriptions, and a search/filter function to catalog outdoor observations.
